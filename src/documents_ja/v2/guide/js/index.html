端末側の戻るボタン Android 端末では、ハードウェア側の戻るボタンが押されたときに、Cordova 側で `backbutton` イベントを発火させます ( **Cordova 側が、このイベントを発火させた** ことに注意が必要です )。このため、`cordova.js` ファイル、または、同等の処理を行うファイル ( Monaca であれば、`loader.js` ) をプロジェクト内で読み込む必要があります。ハードウェアの戻るボタンはブラウザー上では使用できませんが、[Monaca デバッガー](https://ja.monaca.io/debugger.html) を使用すれば、実際の端末上で検証できます。 Onsen UI では、端末側の戻るボタンが押されたときの挙動を、デフォルトで設定している要素もあります。 * ダイアログ : cancelable 属性を指定している場合、ダイアログを閉じます。 * ナビゲーター : ページスタック内にページが置かれている場合、そのページを表示します。 * Splitter : メニューが開いている場合、メニューを閉じます。 条件が満たされない場合、これらの要素は、親要素のハンドラを呼び出し、最終的には、最上位のハンドラ ( ここでは、「 グローバルハンドラ 」 と呼称します ) がアプリを終了させます。 多くのアプリにおいて、上記の対応で十分かと思いますが、Onsen UI ではグローバルハンドラ ( 最終的に、アプリを閉じるハンドラ ) の挙動を、さらにカスタマイズできます。カスタマイズする場合には、[`ons`](/v2/docs/js/ons.html) オブジェクトが提供するメソッドを使用します。 ```javascript // 無効化、または、有効化 ons.disableDeviceBackButtonHandler(); ons.enableDeviceBackButtonHandler(); // 新規ハンドラの登録 ons.setDefaultDeviceBackButtonListener(function(event) { ons.notification.confirm('Do you want to close the app?') // ユーザー側に確認します。 .then(function(index) { if (index === 1) { // OK が押された場合 navigator.app.exitApp(); // アプリを終了します。 } }); )); }); ``` これとは別に、前述の要素 ( dialog 系、navigator、splitter ) と `` 要素では、`onDeviceBackButton` プロパティーが提供されているので、このプロパティーを使用して、戻るボタンのハンドラの挙動を制御することもできます。 ```javascript var myNavigator = document.getElementById('my-navigator'); myNavigator.onDeviceBackButton.disable();
